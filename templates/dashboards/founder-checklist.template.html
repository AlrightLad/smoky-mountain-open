Founder Checklist

The single list of tasks that need a human, not the orchestration agent — money, external dashboard logins, credential pastes, hardware, or a production deploy gated by AMD-018. Each item is rated red (urgent), yellow (would benefit), or green (quality lift), and names exactly who can clear it. Open the walkthrough for exact steps, then Mark complete when done — orchestration auto-verifies and updates this list.

New here? How a developer uses this checklist ↓

This page exists because some work genuinely can't be done by the agent: it needs money, an external dashboard login, a credential paste, hardware/device access, or a production deploy gated by AMD-018. Everything else, the agent just does. When something needs a human, it shows up here.

You don't have to be the Founder. As the team grows, any authorized maintainer can clear most of these. Every open item names who can.

  1. Read the item. Each open card opens with a "Who can do this" line naming the access or role required (for example, "any maintainer with production Firebase deploy access"). Severity signals urgency: red blocks downstream work, yellow would help, green is polish.
  2. Open the walkthrough. Expand the full walkthrough for exact, copy-pasteable steps. You never need to open another file; everything is inline.
  3. Do the steps. Run the command, paste the secret, or toggle the console setting as described.
  4. Mark it complete. Click Mark complete (copy command), paste the copied line into PowerShell, and run it. Orchestration runs the item's verify step automatically: on success the item moves to Recently closed; on failure it shows the verification output so you can see what's still off.

Gated production deploys (Cloud Functions, Firestore rules) carry an AMD-018 gate. For those, the authorization is the Founder's go-ahead and the executor is any maintainer with deploy access — the card spells out both.

This list is always current. It regenerates on every commit and is cross-checked by a freshness sentinel, so what you see matches the live state of the repo. Source of truth: .claude/state/task-queue/founder/.
