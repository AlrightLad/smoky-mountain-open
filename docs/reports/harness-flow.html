layer 3 of 5

The workflow

7 stages · directive to staging

A Founder directive enters at the left, runs through plan → TDD → implementation → review, and exits at the right as a staging deploy ready for Founder visual sign-off. Each stage is an agent or skill; the harness logs cross-stage decisions so subsequent sessions can pick up cold.

stage 1 Directive Founder
stage 2 Plan + spec Orchestrator + Plan agent
stage 3 TDD red tdd-guide
stage 4 Implement Engineer
stage 5 Review code-reviewer + security-reviewer + Critic
stage 6 Verify + smoke PROP-011 verifiers + Playwright
stage 7 Push + staging Engineer (AMD-018 gated)

Two skills bracket every action: outcome-vs-task fires at the START (enumerate ≥3 approaches before the first try), continuation-discipline fires at the STOP (only stop on one of the 7 documented stop conditions). Together they guard against doing-mode failures and false stops.

The audit trail

memory · cron · retrospectives

Every decision lands as a durable artifact. Memory captures cross-conversation knowledge (Founder preferences, project state). Cron telemetry timestamps every hourly action. Retrospectives close each ship with hindsight + foresight.

  • Memory index — user / feedback / project / reference memories ~/.claude/projects/…/memory/MEMORY.md
  • Cron telemetry — hourly action timestamps .claude/state/telemetry/events/<date>.ndjson
  • Heartbeats — watcher + regen status .claude/state/heartbeats/
  • Retrospectives — per-ship 5-component closure .claude/state/retrospectives/
  • Ship progress — per-ship acceptance JSON .claude/state/ship-progress/<ship-id>.json
  • Discussion bubbles — decision-bubble transcripts .claude/state/discussion-bubbles/
  • Stop decisions — daily stop-condition citations .claude/state/stop-decisions/<date>.ndjson
