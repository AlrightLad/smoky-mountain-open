# BL-008: a11y — drawer aria-modal/role="dialog" toggling at Band A

Drawer component does not currently toggle `aria-modal` and `role="dialog"` correctly at Band A (mobile viewport). Defer to future a11y polish ship.

## Resolution (v8.23.27)

Root cause was broader than the captured premise. `_wireSidebar()` in
`src/core/router-sidebar.js` stamped `role="dialog"` + `aria-hidden="true"`
+ `aria-modal="false"` on `#rrSidebar` **unconditionally at every viewport**.
The Band A (720-959px) off-canvas drawer behavior was already correct; the
real defect was that the visible DESKTOP nav rail (>=960px) was permanently
`aria-hidden="true"` + `role="dialog"`, hiding it from and mislabeling it to
screen readers (WCAG 4.1.2 Name/Role/Value, 2.4.1 Bypass Blocks, 1.3.1 Info
& Relationships).

Fix: a `matchMedia("(min-width: 720px) and (max-width: 959px)")` band query
drives `_applyDrawerA11y()`, which sets dialog/modal/hidden semantics only in
the drawer band, sets `aria-label="Primary navigation"` (no dialog role) on
the desktop rail, and clears the role in the <720px CSS-hidden band. Called
on wire, on band change, and from `_openHQDrawer`/`_closeHQDrawer`.

Acceptance criteria:
- [x] Band A drawer sets `role="dialog"` + `aria-modal="true"` when open
- [x] Removes/flips attributes when closed (`aria-hidden=true`, `aria-modal=false`)
- [x] Focus trap engages (pre-existing `_drawerKeydown`, unchanged)
- [x] Smoke covers — `tests/e2e/flows/08-drawer-a11y.spec.js` (5 tests, all pass)
- [ ] VoiceOver / TalkBack on physical device — deferred to Founder device QA
      (Tier 3); runtime ARIA verified headless via `scripts/visual-audit/verify-drawer-a11y.mjs`

Verified before/after with `verify-drawer-a11y.mjs`: desktop went from
`role:dialog, ariaHidden:true, label:"Navigation drawer"` to
`role:null, ariaHidden:null, label:"Primary navigation"`; Band A
closed/open semantics confirmed correct; Band B role cleared.

## Notes

Small but real a11y gap. The captured premise (Band A toggling) understated
it: the desktop rail was the more serious offender. Worth re-running the
verify harness if `router-sidebar.js` drawer wiring is refactored.
