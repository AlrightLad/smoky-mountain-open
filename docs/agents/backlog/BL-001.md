# BL-001: Play Now — hole par/yardage adjustment during live play

**Severity:** P2
**Captured:** v8.3.2 Honey Run hole 10 flag
**Wave/Phase target:** Wave 2
**Status:** Open

## Description

No way to adjust hole par or yardage during live play. This affects FIR (Fairway in Regulation), GIR (Greens in Regulation), and differentials. When the course pars or yardages in the database don't match the actual played course (different tees, scorecard variant, etc.), members can't correct without abandoning the round.

## Files affected

- `src/pages/playnow.js` — live play state and hole rendering
- `src/pages/scorecard.js` — scorecard input flow
- `src/core/handicap.js` — differential computation consumes hole pars

## Dependencies

- W1.S4 Round capture core (canonical round shape stable)
- Design spec for in-round par/yardage edit affordance

## Acceptance criteria for closure

- Member can edit hole par for current hole during live play
- Member can edit hole yardage for current hole during live play
- Edits scoped to this round only (do not affect course master data)
- FIR / GIR / differential recompute correctly after edit
- Smoke covers edit flow

## Last activity

v8.3.2 flagged Honey Run hole 10 issue; deferred to post-Part-B.

## Notes

Course master data is canonical; per-round overrides need their own field on the round document. Cross-surface dependency: any downstream consumer of hole par must read round-override first, then course master.
