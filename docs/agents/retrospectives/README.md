# Retrospectives

Post-push retrospective outputs per POST_PUSH_RETROSPECTIVE.md.

## Naming

Files named `<YYYY-MM-DD>-<short-id>-<ship-id>.md`

Example:
- `2026-06-15-friend-system-W1.S3.md`
- `2026-06-20-bug-triage-listener-activation-W1.I1.md`

## Lifecycle

1. **Generated** — Orchestrator creates after every push, before ship close
2. **Verified** — Critic verifies all 5 components present and accurate
3. **Founder-reviewed** — Founder reads as primary visibility surface between Vision authoring and wave-close grading
4. **Preserved** — never archived or deleted; retrospectives are permanent record of orchestration team work

## Empty state

(No retrospectives at Phase 1 commit. First retrospective fires after Wave Zero Dry-Run passes — that retrospective is proof the protocol works before real ships depend on it.)
