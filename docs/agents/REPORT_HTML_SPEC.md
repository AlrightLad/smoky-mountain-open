# REPORT_HTML_SPEC.md

> **Status:** Governance v8 addition. Locked Founder ratification 2026-05-12.
> **Purpose:** Define HTML report design + technical implementation. Each markdown report from REPORT_TEMPLATES.md pairs with an HTML version. HTML reports are self-contained, work locally + via GitHub raw view.

---

## 0 — Design principles

1. **Self-contained** — each HTML report works when opened locally (file://) OR via GitHub raw view. No backend, no auth, no external data fetch.
2. **Data embedded inline** — telemetry data JSON injected at generation time as `<script>` block; no async loading.
3. **Single dependency** — Chart.js from cdnjs (only external dependency, fallback to embedded if offline). No build step.
4. **Aesthetic match** — dark mode, billiard green + brass accents from PARBAUGHS palette. Matches the AI Engineer Factory reference Founder shared.
5. **Static** — no server-side rendering, no auth, no API calls. Pure HTML+CSS+JS.

### 1.2 Typography

- **Headers:** system-ui sans-serif (Segoe UI, San Francisco, Roboto fallback)
- **Body:** same as headers
- **Numbers + metrics:** ui-monospace, "Cascadia Mono", "JetBrains Mono", Consolas
- **Sizes (mobile-first scale):**
  - h1: 2rem
  - h2: 1.5rem  
  - h3: 1.25rem
  - body: 1rem (16px)
  - small: 0.875rem
  - tiny: 0.75rem

### 1.3 Layout grid

- **Container:** max-width 1400px, centered
- **Card grid:** CSS Grid, `repeat(auto-fit, minmax(320px, 1fr))`, 1.5rem gap
- **Spacing scale:** 0.25rem / 0.5rem / 1rem / 1.5rem / 2rem / 3rem / 4rem

### 1.4 Component styles

- **Section headers:** brass color, uppercase, letter-spaced, 0.75rem
- **Card titles:** primary text, 1.25rem, weight 600
- **Metrics (big numbers):** monospace, 2rem, brass color
- **Metric labels:** secondary text, 0.875rem
- **Tables:** zebra-stripe rows, subtle borders, monospace numbers
- **Progress bars:** thin (4px), brass fill on subtle background
- **Status badges:** rounded pill, moss for success / amber for warning / claret for halt

## 2 — Page structure

Every HTML report follows this skeleton:

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>{report_title} — PARBAUGHS Orchestration</title>
    <link rel="stylesheet" href="../_assets/dashboard.css">
</head>
<body>
    <header class="report-header">
        <div class="container">
            <div class="report-meta">
                <span class="report-type">{REPORT_TYPE}</span>
                <span class="report-generated">Generated {timestamp_iso}</span>
            </div>
            <h1 class="report-title">{report_title}</h1>
            <div class="report-subtitle">{subtitle}</div>
        </div>
    </header>
    
    <main class="container">
        <!-- Report-type-specific sections rendered here -->
        <section class="metrics-grid">
            <!-- Stat cards -->
        </section>
        
        <section class="charts-grid">
            <!-- Chart panels with <canvas> elements -->
        </section>
        
        <section class="tables-grid">
            <!-- Data tables -->
        </section>
    </main>
    
    <footer class="report-footer">
        <div class="container">
            <p>Generated by parbaughs-report-generate from <code>.claude/state/telemetry/aggregates/</code></p>
            <p><a href="../index.html">All reports</a> · <a href="./{report_id}.md">Markdown version</a></p>
        </div>
    </footer>
    
    <!-- Embedded telemetry data -->
    <script id="report-data" type="application/json">
        { /* all report data injected here at generation time */ }
    </script>
    
    <!-- Chart.js (with offline fallback embedded if generation runs offline) -->
    <script src="https://cdnjs.cloudflare.com/ajax/libs/Chart.js/4.4.0/chart.umd.min.js" 
            integrity="sha512-..." crossorigin="anonymous"></script>
    
    <!-- Renderer logic -->
    <script src="../_assets/dashboard.js"></script>
    
    <script>
        document.addEventListener('DOMContentLoaded', () => {
            const data = JSON.parse(document.getElementById('report-data').textContent);
            ParbaughsReports.render('{report_type}', data);
        });
    </script>
</body>
</html>
```

## 3 — Chart specifications

### 3.1 Charts per report type

**Dashboard report:**
- Doughnut chart: token consumption by role (current week)
- Line chart: 7-day token trend
- Stacked bar: cycle outcomes (last 7 days)
- Gauge-like progress bar: weekly budget consumption

**Daily report:**
- Stacked bar: tokens by activity (this day, per cycle)
- Line chart: cycle outcomes timeline (this day)
- Doughnut: handoffs by scenario

**Weekly report:**
- Line chart: weekly cost trend (last 4 weeks)
- Bar chart: ships completed per week (rolling 12 weeks)
- Stacked bar: tokens by role per day (this week)
- Doughnut: proposal lane distribution

**Ship report:**
- Stacked bar: tokens by phase
- Doughnut: tokens by role
- Timeline: cycle waterfall for this ship
- Bar chart: discussion bubbles per day during ship

**Wave report:**
- Gantt chart: ship timeline (using Chart.js with custom config)
- Stacked area: cost accumulation over wave duration
- Bar chart: ships ranked by token cost
- Trend lines: code metrics over wave (lines, coverage, tests)

**Quarterly report:**
- Multi-series line: tokens per week
- Stacked bar: cost by wave
- Trend lines: velocity, cost, quality (composite metric)

### 3.2 Chart common config

All charts use:
- Dark theme: background transparent, text in `--text-primary`/`--text-secondary`
- Color palette: `--chart-1` through `--chart-6` series colors
- Grid lines: `--border-subtle` for axes
- Tooltips: dark with brass border
- No legend below if space tight (use side legend on wide layouts)
- Responsive: resize on window change

## 4 — Interactivity features

### 4.1 Standard interactivity (all reports)

- **Hover tooltips on charts** — show exact values
- **Sortable tables** — click column headers to sort
- **Expandable sections** — collapse/expand cards
- **Time range filter** (where applicable) — adjust date window
- **Theme toggle** — light/dark (defaults to dark)
- **Drilldown links** — click ship → ship report, click wave → wave report

### 4.2 Per-report interactivity

**Dashboard:**
- "Refresh now" button (re-reads embedded data, but data is static once page loaded — this is mostly a visual indicator of staleness)
- Quick links to all other reports

**Weekly + Wave:**
- Filter by lane (proposals charts)
- Filter by role (token charts)
- Toggle between absolute and percentage views

### 4.3 NOT implemented (out of scope for v8)

- Real-time data refresh (page is static; reload to get latest)
- Server-side filtering (all filtering is client-side over embedded data)
- Edit/comment on reports (reports are read-only)
- Export to PDF (use browser print-to-PDF if needed)
- Email alerts (no email integration)

## 5 — Performance budget

Each HTML report must:
- Load + render in <2 seconds on standard laptop
- File size <500 KB (data + HTML + CSS combined, Chart.js loaded from CDN)
- Mobile-render correctly at 375px viewport (your phone scenario)
- Work offline once loaded (Chart.js may need to be embedded if offline mode required)

---

## 6 — Accessibility

- Semantic HTML: proper heading hierarchy, table headers, ARIA labels
- Color contrast: WCAG AA minimum on all text
- Keyboard navigation: all interactive elements focusable
- Screen reader: chart data also available as tables (parallel rendering)
- Dark mode: respects `prefers-color-scheme`

## 8 — Generation flow

Per `parbaughs-report-generate` skill:

```
1. Read aggregate telemetry data
2. Compute report-specific metrics  
3. Render markdown report (write to .claude/state/reports/)
4. Generate HTML:
   a. Read template skeleton (docs/reports/_assets/template.html)
   b. Inject data as JSON in <script id="report-data">
   c. Inject report-specific HTML sections
   d. Write to docs/reports/{type}/{name}.html
5. Update docs/reports/index.html with link to new report
6. Commit both .md and .html files
```

## 10 — Access patterns

Founder accesses reports via:

### 10.1 GitHub web UI
- Browse to `docs/reports/index.html` in repo
- GitHub renders the HTML inline (or click "Raw" for direct view)
- Click links to drill into specific reports

### 10.2 Local clone
- `git pull` to get latest
- Open `docs/reports/index.html` in browser
- Full interactive experience

### 10.3 Direct file URLs
- Bookmark: `https://github.com/AlrightLad/smoky-mountain-open/blob/main/docs/reports/dashboard.html`
- Or raw view: `https://raw.githubusercontent.com/AlrightLad/smoky-mountain-open/main/docs/reports/dashboard.html` (use htmlpreview.github.io if needed)

### 10.4 NOT public deployment
- Reports are NOT deployed to GitHub Pages (the public site is the production app)
- Reports stay in repo, accessible to Founder via repo access
- Future option: separate private GitHub Pages branch (requires GitHub Pro)
