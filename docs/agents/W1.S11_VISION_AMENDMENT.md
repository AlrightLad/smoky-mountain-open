## W1.S11 — Feed + Activity (amended Vision)

> The Feed + Activity ship lands the platform's social communication and activity surfaces unified under one architecture — Feed cards (round posts, Chip posts, auto-posted activity events) + League Pulse activity stream + Activity detail view from masthead avatar/Profile sheet. Chip is the canonical short-form social post format (280 character limit, member-authored, covers standalone text, photo posts, round shares, party-game shares, championship celebrations) — ALL social posts are Chips, single composer pattern, consistent feed surface treatment across League scope and the public/community scope. The public/community scope (orchestration team picks the name from "Community", "The Range", "Open Play" or similar candidates via decision-bubble process consulting locked country-club tone) replaces the design-spec 3-tab scope rail with the locked 2-tab structure — League scope + the named public/community scope, eliminating "Network" tab. Public scope shows: league rounds visible to public, Chips from members on public posting, cross-league friend posts per memory #28 friend system layer, public round shares, photo posts. Tone between low-frequency rewarding and high-frequency engagement — feed should feel like worthwhile reading not addictive scroll. Activity is "everything Nick has been up to" chronological view, accessible from member's profile or masthead avatar tap. Auto-posted activity events include: round finalized, championship awarded (per W1.S9), friend joined league, wager initiated/settled (per W1.S6), party game claimed (per W1.S7), course added (per W1.I6), Trophy earned. Member-authored Chips intermingle with auto-posted events in reverse-chronological order, with live rounds pinned to top per locked W2.S1 Home Vision Section A pattern. Founding 20 are auto-friended per memory #28 so the initial feed populates with content even before community-uploaded content accumulates. P10 loop-and-verify discipline applies. Engineer and Critic verify via Playwright + iOS simulator + HQ Web that Chip composer enforces 280 char limit at the input layer (UI prevents > 280 typing, additional defense at Firestore security rule layer rejecting longer Chips), that all Chip post types render correctly in feed (text only, photo attached, round-share with referenced round metadata, party-game share with attribution, championship celebration), that scope rail toggles correctly between League and public/community with content correctly filtered per scope, that auto-posted activity events fire correctly from all source ships (W1.S4 round finalize, W1.S9 championship award, W1.S3 friend join, W1.S6 wager events, W1.S7 party game events, W1.I6 course add), that live rounds pin correctly per locked W2.S1 pattern, that cross-league friend posts surface correctly in public scope without exposing private member data (Firestore security rules enforce visibility), that activity detail view from masthead avatar tap correctly shows "everything member has been up to" chronologically, and that founding-20 auto-friended baseline produces populated feed at platform launch. End User sub-agent sweep verifies Beginner profile understands Chip composer immediately (no jargon, clear character counter, intuitive media attachment), Mid-handicap and Scratch profiles verify feed tone matches country-club editorial voice rather than addictive social media patterns, Lone Wolf profile verifies public scope works without league affiliation (lone members can post Chips and see public content), Commissioner profile verifies no elevated feed authority (commissioners are regular Chip authors like everyone else). Per locked Vision discipline: deep, thorough research before code lands; fault-tolerant design with explicit revert path if Chip post type integration disrupts existing feed architecture.

## What changed from original W1.S11 Vision

1. **Chip post type integration** — Chip is now the canonical short-form social-post format covering text, photo, round-share, party-game-share, championship celebration. All social posts are Chips; one composer; consistent feed surface.

2. **280 character limit locked** — per Founder decision during Vision authoring session.

3. **2-tab scope rail** — League scope + the orchestration-team-named public/community scope. "Network" tab eliminated. Spec-original 3-tab assumption (League/Network/All) replaced.

4. **Public scope content composition explicitly enumerated** — league rounds visible to public + Chips + cross-league friend posts + public round shares + photo posts.

5. **Activity surface clarified** — masthead avatar tap → Activity detail view = "everything member has been up to" chronologically.

6. **Founding-20 auto-friended baseline** per memory #28 — produces populated feed at platform launch.

7. **Scope-naming decision delegated to orchestration team** — name picked via bubble process during implementation; not blocking Vision ratification.
