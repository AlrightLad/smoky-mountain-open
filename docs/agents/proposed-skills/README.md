# Proposed Skills

Skill proposals pending Founder approval before commit to `.claude/skills/`.

Per `.claude/settings.json` hook 5: `.claude/skills/` requires `SKILL_APPROVAL.md` token. Proposals draft here; on Founder approval, skill content moves to `.claude/skills/` along with approval token.

## Why proposals live here first

Skills modify agent behavior. Adding a skill without Founder ratification could shift agent decision patterns in ways that drift from Founder intent. The approval gate keeps Founder in the loop.

## Proposal protocol

1. Orchestrator drafts skill content (Tier 1 graduated autonomy)
2. File saved here as `proposed-<skill-name>.md` with intended skill definition
3. Founder reviews at next retrospective (or earlier if Orchestrator requests)
4. Founder ratifies → skill moves to `.claude/skills/<skill-name>/SKILL.md` + `SKILL_APPROVAL.md` token committed
5. Founder rejects → proposal stays here with rejection rationale, or is deleted
