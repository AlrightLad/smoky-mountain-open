## Vision (ratified — DO NOT MODIFY)

W1.S1 codifies the design system as the foundation every subsequent Wave 1 ship consumes. Locks the country-club editorial palette (`--cb-felt`, `--cb-chalk`, `--cb-ink`, `--cb-brass`, `--cb-mute`, `--cb-claret`, `--cb-moss` base palette plus 6 new tokens per W2.S0 reveal — `--cb-ink-soft`, `--cb-ink-faint`, `--cb-mute-soft`, `--cb-mute-faint`, `--cb-claret`, `--cb-moss` as full additions to shared palette consumed by both HQ Web AND Clubhouse Mobile per locked decision G1). Typography hierarchy: Fraunces (display + member identity), Inter (body + UI), JetBrains Mono (stats + meta). Spacing scale (4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 / 96). Hairline rules + brass underline + chalk-on-felt patterns. SVG icon library (no emojis per locked LB1 — high-class emblems only). Reduced-motion respect. Sunlight mode auto-detection per `prefers-color-scheme` + manual override via Settings. AAA contrast minimum on critical paths. Theme-aware inline SVG pattern per memory #16 — `<svg style="color: var(--cb-brass)">` + `stroke="currentColor"` for plot elements. Token traceability: NO raw hex / px / ms values in production source — every value via CSS variable. Build configuration ensures Vite-split bundle includes token definitions in shared CSS imported by every page. CSS token usage audit per memory #6 — grep `var(--*)` across `src/pages/` + `src/core/` confirms every JS inline style reference resolves to a declared token. Per locked Vision discipline: deep research before code lands; fault-tolerant design with explicit revert path if token system breaks production consumers.
