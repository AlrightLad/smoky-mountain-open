### In scope

Per ROADMAP.md W1.S1: "Design system codification (Wave 2A spec → consumable CSS variables + component classes + JS helpers)".

Foundation tier ship — everything in Wave 1 consumes its output. **Must ship before W1.S2-S14** (per ROADMAP.md strict ship order: `W1.S1 → W1.S2 → ...`).

Token codification from `docs/CLUBHOUSE_SPEC.md`:
- **§1 Palette** — surface + ink + brass + status palette + semantic aliases → `src/styles/base.css` `:root` token block (current Clubhouse tokens audited + extended per spec)
- **§2 Type Scale** — Fraunces (display) + Inter (UI) + JetBrains Mono (technical) stacks + scale tokens (--text-2xs through --text-hero-xl per CLAUDE.md current)
- **§3 Spacing & Layout** — base unit + scale + layout tokens + safe-area handling + mobile-specific elevation
- **§4 Motion Vocabulary** — duration tokens + easing tokens + composite patterns + reduced-motion handling
- **§5 Gesture Pattern Library** — JS gesture helpers (deferred to Wave 3 M-ships per spec § 5.1 position, but token-level prep may land here)
- **§6 Accessibility Framework** — contrast verification, outdoor-use scenarios, AA/AAA thresholds

JS helpers per W1.S1 scope:
- Per CLUBHOUSE_SPEC §4 motion patterns: `src/core/animate.js` already exists; extend with token-driven motion API
- Token-resolution helpers for SVG (per `theme-aware SVG pattern` in ENGINEER.md skill): `var()` doesn't resolve in SVG presentation attrs; helper to inject `currentColor` pattern

Component class extraction per CLAUDE.md "Design System":
- Confirm semantic surface / text / border / accent aliases ALL covered (audit current `--surface-primary/secondary/...`, `--text-primary/...`, etc.)
- Audit legacy aliases (`--bg`, `--bg2/3/4`, `--cream`, `--muted`, etc.) — preserve or sunset per Vision direction

### Out of scope

- Per-page component application (Wave 2 design coherence pass; per-page mocks)
- Mobile-specific patterns from CLUBHOUSE_SPEC § 5 gesture library (Wave 3 M1+ scope)
- Wave 2A § 3 crisis banner UX (W1.I5 ship per ROADMAP.md)
- HQ chrome refresh / masthead redesign (W1.S2 ship per ROADMAP.md)
- Visual regression baseline (Wave 1 → 2 gate criterion; tools land in W1.S1 as enabler)

### Cross-surface dependencies (Criterion 12)

- **Every page** in `src/pages/` consumes CSS tokens (45 page files per Phase 1 audit + 19 core files)
- **`var(--*)` reference count: ~1,800** per CLAUDE.md (verify current via grep)
- **`base.css` is the source of truth** for token declarations; `components.css` and inline `style="..."` consume
- **Per CLAUDE.md "Visual exceptions":** share card template (`.pbs-*` rules) intentionally uses hex literals (html2canvas constraint); hole-dot + calendar-dot constants intentional; boot-fallback `<style>` in index.html intentional

Token changes ripple through every page. **Migration-window pattern mandatory** if any token is renamed (additive-only by default; new alongside old; sunset old in follow-up ship).

## Scalability Architecture (per Q44 Lock 3)

### Free-tier first
- No paid services
- Bundle size impact: new tokens add ~5KB to base.css; expected to be offset by deletion of legacy aliases per audit
- No API calls

### 10x scaling plan
- Tokens scale O(1) regardless of member count
- Token count growth is bounded by spec (CLUBHOUSE_SPEC § 1 lists finite token table)
- Component-class growth follows feature growth, not member-count growth

### Cost projection
- Zero. Pure CSS + JS.
