### Q1 — Phase sequencing (proposed answer)

**Decision: Per the two-ship split in Q5, sequencing is:**

**S1.b.1 (Foundation, infra-ship, no AMD-012 smoke gate):**
1. Palette value refinement — OKLCH derivation; replace 4 placeholders; add `--cb-felt-soft` + `--cb-felt-deep` per Q7
2. Token traceability tooling — lint rule + round-trip extension
3. Vite-bundle config verification — confirm tokens reach every page first-paint
4. Reduced-motion completion audit — every animation in `components.css` verified
5. Ship-close: round-trip green + traceability lint clean + motion audit complete

**S1.b.2 (Member-visible, design-ship, AMD-012 smoke gate applies):**
1. SVG icon inventory — grep emoji + icon-glyph usages in `src/pages/` + `src/core/`
2. SVG icon library implementation + theme-aware pattern (memory #16)
3. Sunlight mode HQ spec authoring + implementation
4. AAA contrast audit (axe-core) + remediation
5. Ship-close: smoke per AMD-012 + visual + functional + round-trip green

**Rationale:**
- The original 9-phase numbering in this doc's Phase boundaries section conflated INDEPENDENCE (can ship parallel) with FOUNDATION-VS-VISIBLE (audience boundary). The Q5 audience split makes phase ordering fall out cleanly: foundation first (S1.b.1), member-visible second (S1.b.2).
- Within S1.b.1, palette refinement comes first because every other phase depends on stable token values. Traceability tooling follows because it must be in place before consumers start adopting tokens. Vite-bundle config + reduced-motion audit are independent and can interleave.
- Within S1.b.2, SVG inventory must precede icon library implementation. Sunlight mode requires spec authoring before implementation (per honest delta #4 in prior report). AAA audit comes after visual treatments are in final state.

**Alternatives considered + dismissed:**
- (a) Strict independence-ordered sequencing — dismissed because it scatters audience-related work across phase boundaries, making the AMD-012 smoke gate harder to apply cleanly.
- (b) Pure linear sequencing (1 → 9) — dismissed because the audience boundary is real; conflating infra + member-visible inflates AMD-012 scope unnecessarily.

**Failure modes:**
- If S1.b.1's palette refinement produces values incompatible with existing partial UI usages, visual regression. Mitigation: Phase 1 includes a quick regression check via `npm run dev` + browser visual inspection on key pages.
- If S1.b.2's SVG inventory surfaces >500 icon-roles, S1.b.2 expands and may need further split. Mitigation: per-category SVG batches (navigation / status / action / decorative) with independent ship-close per batch.
