## Scalability Architecture (per Q44 Lock 3)

### Free-tier first
- No new paid services
- No new external API calls
- Schema unchanged (additive at most; current scope is all rendering-layer)

### 10x scaling plan
- Current load: 20 members; renders all in one page (no pagination)
- 200 members: still feasible in single render (40KB DOM); may need pagination at 500+
- 2000 members: pagination + search-as-you-type mandatory
- Migration path: pagination added in W2 if member count grows mid-Wave

### Cost projection
- Firestore reads: members list = 1 `.get()` per page load; ~20 docs at current scale
- No write changes (E1 username lock reduces writes if anything)
- No Cloud Function impact
