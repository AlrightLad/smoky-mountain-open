# Monthly architecture strategic review

One file per ISO month: `<YYYY-MM>.md` (e.g. `2026-05.md`). Generated
by the Architecture agent's monthly cycle (8-12 hour runtime, anchored
last Sunday of the month).

## Required sections

```markdown
# Monthly strategic — <YYYY-MM>

## Substrate maturity assessment
(what got more mature this month? what regressed?)

## Multi-week pattern synthesis
(patterns spanning 4 weeks of daily/weekly outputs;
 things invisible at weekly cadence)

## Architecture evolution roadmap
(what should the system look like in 30 days? 90? 6 months?
 specific milestones)

## Strategic recommendations
(big-picture moves — substrate amendments, new agents,
 capability gaps to fill)

## Free-replication progress
(which paid services moved to free this month?
 which paid services still on the radar?)

## Substrate health KPIs
- Amendments ratified vs proposed
- Recommendations from architecture agent: ratified vs proposed
- False-stop incidence rate
- Approval pipeline reliability
- Agent coordination effectiveness (cross-queue traffic)
- Free-replication count

## Sources cited
```

The monthly review is where compounding effects show. Daily is the
moment, weekly is the iteration, monthly is the trajectory.
