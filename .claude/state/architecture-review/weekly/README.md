# Weekly architecture deep review

One file per ISO week: `<YYYY-WW>.md` (e.g. `2026-20.md`). Generated by
the Architecture agent's weekly cycle (3-5 hour runtime, anchored
Sunday).

## Required sections

```markdown
# Weekly deep review — <YYYY-WW> (week of <Sunday date>)

## Executive summary
(one paragraph; what changed, what mattered, what's coming)

## Step 1: State the problem(s)
(what's actually happening across the week)

## Step 2: Hypotheses
(at least 3 hypotheses per major friction, with evidence)

## Step 3: Ecosystem research
(this week's Anthropic releases, community patterns,
 relevant papers — with primary-source links)

## Step 4: Free alternatives evaluated
(any paid services considered + free path)

## Step 5: Recommendations
(priority-ranked, with rationale, expected outcome,
 validation, cost, rollback)

## Step 6: Second-order effects
(what each recommendation affects elsewhere)

## Step 7: Sources cited
(bibliography)

## Memory consolidation
(dream-skill output for the week)

## Self-evaluation
(prior week's recommendations: ratified/applied/outcome
 vs prediction — refines methodology)
```

Reads daily/ files for the week as input. Cross-references stop-decisions,
error-backlog, agent activity, test-qa cycles, security findings.
