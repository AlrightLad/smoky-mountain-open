# W1.S1 ship plan

## Spec source (Principle 1 — diagnose to evidence)

The ratified W1.S1 VISION at `docs/agents/ships/W1.S1.md` is broad
Clubhouse system codification: `--cb-*` tokens (felt / chalk / ink /
brass / mute / claret / moss), 6 new W2.S0 tokens, AAA contrast,
sunlight mode auto-detection, full SVG icon library, theme-aware
inline SVG pattern, reduced-motion, token traceability across
`src/pages/` and `src/core/`. The vision's "acceptance criteria
placeholders" line explicitly defers full enumeration to the team at
pre-flight.

That vision is NOT detailed enough to drive a single ship's scope —
it spans the entire Clubhouse design system (mobile + HQ) and would
consume tens of millions of tokens to ship completely. It is
operational direction, not actionable scope.

Per Founder's W1.S1 authorization decision tree ("If NOT detailed
enough, the spike comparison.md + the 3 spike-surfaced primitives
become the spec"), this iteration's spec source is:

- **Primary:** `.claude/state/design-spike/comparison.md` (Approach A
  PARBAUGHS native recommendation, locked by Founder)
- **Surfaced gaps:** the 3 primitives spike identified as missing from
  `_assets/design-system-components.css`: `.pb-avatar`, `.pb-list` +
  `.pb-list-row`, `.pb-trend-delta`

This is W1.S1 **iteration 1**. Broader Clubhouse-system codification
(--cb-* tokens, SVG icon library, sunlight mode, AAA paths) is a
follow-on ship — likely W1.S1.b or rolled into W1.S2/S3 scope when
those ships' Wave 2A specs surface.

## Primary approach — Plan A (orchestration-dashboard 3 primitives)

Extend `docs/reports/_assets/design-system-components.css` with the
3 spike-surfaced primitives, extend the `design-system.html`
showcase, extend `tests/round-trip-test.py` to enforce
token-discipline on the new primitives, commit atomic.

## Rollback strategy (concrete commands)

If shipped but post-hoc regression is discovered:

```bash
# Identify ship-closing commit
git log --oneline | grep "W1.S1 ship close"

# Revert that single commit (creates new revert commit)
git revert <ship-closing-sha> --no-edit

# Re-run regen to refresh dashboards
bash scripts/regen-all.sh

# Verify round-trip
PYTHONIOENCODING=utf-8 python tests/round-trip-test.py
```

Risk-of-rollback is LOW because the changes are additive: no
existing primitive declarations modified; no existing CSS class
renamed; no JS contract changed. Removing the new primitives
reverts to pre-W1.S1 design-system state cleanly.
